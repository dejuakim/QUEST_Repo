# PRT(Peer Review Template)
- [X]  **1. 주어진 문제를 해결하는 완성된 코드가 제출되었나요?**
    - 정답 숫자를 랜덤으로 뽑고 5자리 숫자를 입력하여 strike와 ball로 출력되게 하였습니다.
    
- [X]  **2. 전체 코드에서 가장 핵심적이거나 가장 복잡하고 이해하기 어려운 부분에 작성된 
주석 또는 doc string을 보고 해당 코드가 잘 이해되었나요?**
    - 해당 코드를 입력하는 과정에서 중복된 숫자가 들어가거나 9 이상의 숫자가 입력되면 재입력하도록 했습니다.
    - 주석을 통해 해당 코드의 기능을 잘 설명하였고 코드 이해가 잘 되었습니다.
            
- [ ]  **3. 에러가 난 부분을 디버깅하여 문제를 해결한 기록을 남겼거나
새로운 시도 또는 추가 실험을 수행해봤나요?**
    - 문제와 해결 과정에 대한 기록은 없습니다.
        
- [ ]  **4. 회고를 잘 작성했나요?**
    - 주어진 문제를 해결한 완성된 코드만 확인할 수 있었고 배운점과 아쉬운점, 느낀점 등은 기록되지 않았습니다.
        
- [X]  **5. 코드가 간결하고 효율적인가요?**
    - 코드가 간결하고 함수를 잘 사용하였습니다.
    - 주석을 통해 잘 정리하였으며 가독성이 좋아 코드를 이해하는데 어려움이 없습니다.


코드 리뷰를 통해 주석과 docstring의 중요성을 다시 한 번 느꼈습니다.  
코드가 제대로 작동하는 것도 중요하지만 코드의 의도를 글로 정리해 두는 것이  
그걸 이해하고 유지보수하는 데 큰 도움이 된다는걸 알게 되었습니다.

특히 한 번에 이해하기 어려운 부분일수록 주석과 docstring을 통해 흐름을 설명해 두면  
이후 디버깅이나 기능 확장이 쉬워질 것이라고 느꼈습니다.

아쉽게도 문제를 해결하는 과정에서 에러가 수차례 발생하였으나 이를 해결한 기록을 남기지 못했습니다.
다음 과제에서는 단순히 정답 코드를 제출하는 데 그치지 않고, 문제 해결 과정에서 겪은 시행착오와
고민을 함께 기록해 학습 과정을 더 명확히 남기고자 합니다.

또한 배운 점과 느낀 점을 회고로 정리하는 습관을 통해 코드를 더 간결하고 효율적으로 개선할 수 있도록 노력해야겠습니다.
